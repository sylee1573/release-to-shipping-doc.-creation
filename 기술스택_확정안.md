## 레이어별 스택 선정표

|레이어|선정 기술|선정 이유|대안 검토 후 기각|
|-|-|-|-|
|백엔드 프레임워크|FastAPI (Python 3.11+)|비동기 처리, 자동 Swagger, Claude Code 친화|Flask (동기 처리 한계)|
|AI 파싱 (복잡)|Claude API — Sonnet 4.5|문서 이해·신뢰도 계산 정확도|GPT-4o (비용 대비 우위 없음)|
|AI 파싱 (단순)|Claude API — Haiku 4.5|비용 1/5\~1/10|Sonnet 단일 사용 (비용 과다)|
|PDF 추출|pdfplumber|텍스트 PDF 정확도, 무료|PyMuPDF (라이선스 제약)|
|Excel 생성|openpyxl|양식 제어, 셀 스타일, 무료|xlsxwriter (읽기 불가)|
|데이터베이스|PostgreSQL|RLS 멀티테넌트, 안정성|SQLite (단일 테넌트 한계)|
|스케줄러|APScheduler|FastAPI 내장 통합, 경량|Celery (과한 복잡도)|
|알림|Solapi|카카오 알림톡 + SMS 국내 특화|AWS SNS (국내 알림톡 미지원)|
|프론트엔드|React 18 + Vite + TypeScript|상태관리 필요, Claude Code 친화|Vue (생태계 규모 차이)|
|CSS|Tailwind CSS|빠른 스타일링, 파일 없이 개발|MUI (번들 크기 과다)|
|서버 상태|React Query|API 캐싱, 로딩/에러 처리 자동화|SWR (기능 제한)|
|백엔드 호스팅|Railway.app|PostgreSQL 번들, 자동배포, $5 시작|AWS (초기 설정 복잡도)|
|프론트 호스팅|Vercel|CDN 자동, 무료 플랜, GitHub 연동|Netlify (동일 수준, 친숙도 우위)|
|컨테이너|Docker|설치형 패키징 옵션 대비|—|
|CI/CD|GitHub Actions|Railway/Vercel 자동 연동|Jenkins (과한 복잡도)|
|에러 모니터링|Sentry|무료 티어, FastAPI/React 공식 지원|Datadog (비용 과다)|
|개발 도구|Claude Code|코딩 배경 없이 AI 주도 개발|Cursor (협업 기능 부족)|

## 과금 구조 (확정)

|항목|내용|
|-|-|
|과금 방식|월별 사용량(처리 건수) 기반|
|청구 방법|월말 인보이스 발행 → 이메일 발송|
|수금 방법|고객사 계좌이체 (업계 표준)|
|세금계산서|홈택스 또는 더존에서 수동 발행|
|PG 연동|없음 (자동결제 불필요)|
|관리 도구|관리자 페이지 내 월별 고객사 사용량 리포트|

## 미납 자동 처리 타임라인

```
납부기한(D+0)
    │
D+30 ─── 1차 경고 (이메일 + 카카오 알림톡 / 담당자)
    │
D+37 ─── 2차 경고 (이메일 + 카카오 알림톡 / 담당자 + 대표)
    │
D+44 ─── 중단 예고 ("내일 서비스가 중단됩니다")
    │
D+45 ─── 서비스 자동 중단 (기능 잠금, 데이터 보존)
    │
입금확인 ── 관리자 수동 복구
```

## 파싱 비용 통제 전략

```

2단계 분리:
  pdfplumber 텍스트 추출 (무료)
      ↓
  추출 텍스트만 Claude 전달 (토큰 최소화)
      ↓
  복잡 파싱 → Sonnet 4.5
  단순 정규화 → Haiku 4.5 (비용 1/5)

```
